# Decision: Consolidate completed research progress and Phase 2 results

- Date: 2026-07-14
- Intended commit subject: `Document completed pruning and repair experiments`
- Status: Accepted as the current project record

## Problem And Decision

The repository contained the implementation and protocol for Phase 2, but its completed compact
results existed only on the GPU host. The root README also described Phase 2 as current work and
did not clearly state that 8-, 12-, and 16-block routes had already been evaluated. This made it
easy to propose repeating completed experiments and obscured the difference between promising
development evidence and a publication-ready result.

The decision is to synchronize only compact Phase 2 evidence, add one canonical status report,
and update the top-level documentation to state the completed work, exact results, failed ideas,
claim boundaries, and next unresolved experiment.

## Alternatives Considered

- Commit all Phase 2 predictions and bridge tensors. Rejected because those artifacts are large,
  reproducible, and intentionally excluded from Git; compact aggregates preserve the evidence
  needed for review.
- Report only the best bridge rank. Rejected because the failed and harmful ranks are essential to
  the conclusion that feature reconstruction did not reliably recover behavior.
- Describe the next step as another 8/12/16-block sweep. Rejected because those budgets were
  already evaluated and showed large losses.
- Evaluate the external held-out set immediately. Rejected because interaction-aware route search
  and method selection are not frozen; looking now would turn the held-out set into development
  data.
- Claim that task-aware pruning is novel. Rejected because related work includes task-specific
  encoder pruning. The defensible open question is capability-level causal mapping and selectable
  routes in a generative VLM vision encoder.

## Limitations, Risks, And Unsupported Claims

- Phase 1 and Phase 2 are development-set experiments with route-selection bias.
- The repair is SCP-inspired; it is not an implementation or reproduction of Short-LVLM SCP.
- Lower feature-space relative L2 error does not establish recovered computation.
- Results cover one Qwen model and one RTX 4090, not a second architecture or edge device.
- Identity replacement does not yet create a serialized compact checkpoint.
- The sealed external benchmark may still overlap model pretraining despite image-level dedup.
- This commit does not establish held-out generalization, edge deployment, globally optimal routes,
  or a publication-ready contribution.
